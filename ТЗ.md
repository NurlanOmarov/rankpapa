# Техническое задание: RankPapa — Сервис накрутки поведенческих факторов Google (Казахстан)

> **Версия:** 1.0  
> **Дата:** 31.03.2026  
> **Ориентир UI/UX:** transfer-astana.com  
> **Аналог функционала:** seopapa.com (адаптация под Google + KZ)

---

## 1. Общее описание

**RankPapa** — SaaS-сервис для продвижения сайтов в Google.kz за счёт накрутки поведенческих факторов (ПФ). Каждый «визит» имитирует реального казахстанского пользователя: казахстанский IP (residential/mobile прокси), уникальный браузерный fingerprint, человекоподобное поведение на сайте (скролл, клики, dwell-time).

Сервис рассчитан на два профиля использования:
1. **Накрутка ПФ** — нагон визитов из Google-поиска по ключевым словам клиента.
2. **Трекинг позиций** — автоматический мониторинг позиций сайта и конкурентов в Google.kz.

В будущем платформа масштабируется под другие страны (RU, UA, UZ и т.д.) — архитектура должна это предусматривать.

### 3.2 Личный кабинет клиента

#### 3.2.1 Дашборд
- Сводка по всем активным кампаниям (визиты сегодня / за месяц, динамика позиций)
- Уведомления о смене позиций, окончании баланса

#### 3.2.2 Управление сайтами
- Добавление домена
- Привязка ключевых слов (до N по тарифу)
- Настройка гео (город: Алматы / Астана / Шымкент / Весь Казахстан)
- Настройка типа трафика (десктоп / мобильный / смешанный)
- Настройка дневного лимита визитов
- Статус кампании (активна / пауза / архив)

#### 3.2.3 Накрутка ПФ
- Расписание запусков (часовые диапазоны)
- Ramp-up режим (плавный старт: +X% в день от базового объёма)
- Настройка dwell-time (мин / макс секунд на сайте)
- Настройка глубины сессии (кол-во страниц за визит)
- Статистика: визиты, CTR, средний dwell-time, провалы (отказы)

#### 3.2.4 Трекинг позиций
- Добавление ключей для мониторинга
- Частота проверки: ежедневно / еженедельно
- Мониторинг конкурентов (до N доменов по тарифу)
- Граф истории позиций
- Email/Telegram уведомления при смене позиции на ±N мест
- Экспорт в CSV / PDF

#### 3.2.5 Отчёты
- Сводный месячный отчёт (PDF, брендированный)
- Детальный лог каждого визита (ключ → позиция клика → время на сайте)

#### 3.2.6 Биллинг
- Баланс пополнения (Kaspi, карты Visa/MC, крипто)
- Автоматическое списание по тарифу
- История транзакций
- Счёт / акт для юрлиц

### 3.3 Панель администратора

- Список клиентов, их тарифы, баланс, статус
- Управление тарифными планами
- Мониторинг очереди задач (BullMQ)

## 5. Техническая архитектура

### 5.1 Стек технологий

| Компонент | Технология |
|-----------|-----------|
| Frontend (лендинг + ЛК) | Vue 3 + Vite + Tailwind CSS |
| Backend API | Node.js + TypeScript + Fastify (или NestJS) |
| База данных | PostgreSQL |
| Кеш / очередь | Redis + BullMQ |
| Браузерная автоматизация | Playwright + rebrowser-patches |
| Fingerprints | `playwright-with-fingerprints` (bablosoft FingerprintSwitcher) или Camoufox (open-source) |
| Прокси | PROSOX API (текущий trial) + ротационные KZ residential/mobile |
| Мониторинг позиций | Playwright SERP-парсер Google.kz |
| Инфраструктура | VPS Ubuntu 22.04, Docker Compose (MVP), Kubernetes (v2) |
| Деплой | GitHub Actions → Docker → VPS |
| Мониторинг сервиса | pm2 / systemd, Grafana + Prometheus |

### 5.2 Схема работы накрутки ПФ

```
Клиент создаёт задачу (сайт + ключ + лимит)
         ↓
Backend помещает задачу в Redis-очередь (BullMQ)
         ↓
Worker берёт задачу
         ↓
Запрашивает fingerprint из пула (FingerprintSwitcher / Camoufox)
         ↓
Получает прокси KZ из PROSOX API (Казахстан, residential)
Синхронизирует: timezone=Asia/Almaty, locale=ru-KZ, geo=Алматы
         ↓
Запускает Playwright-браузер с этим профилем
         ↓
Открывает google.kz → вводит ключ → находит сайт → кликает
         ↓
Human-like поведение: скролл (Bezier), случайные клики, 60–300 сек dwell
         ↓
Закрывает браузер, логирует результат в PostgreSQL
         ↓
Статистика обновляется в ЛК клиента в реальном времени
```

### 5.3 Схема работы трекинга позиций

```
Cron-задача (ежедневно в заданное время)
         ↓
Для каждого ключа → Worker
         ↓
Открывает google.kz/search?q=ключ&gl=kz&hl=ru
через KZ-прокси + fingerprint (без dwell, только парсинг)
         ↓
Парсит ТОП-100 результатов: позиция, URL, заголовок
         ↓
Сохраняет в PostgreSQL (история позиций)
         ↓
Если позиция изменилась на ±N → уведомление клиенту
```

### 5.4 Интеграция PROSOX API

Сервис прокси: **PROSOX** (prosox.com)  
Текущий пакет: Premium SX 500MB, Казахстан, активен до 30.04.2026

Параметры генерации прокси:
- **Страна:** Kazakhstan
- **Формат:** `login:password@host:port`
- **Авторизация:** Логин/Пароль
- **Ротация:** настраивается (для трекинга — без ротации, для ПФ — ротация каждые 5–10 запросов)

Интеграция:
```typescript
// proxy.service.ts
interface ProxoxConfig {
  apiKey: string; // из .env
  packageKey: string; // c27970d0808a7f824117
  country: 'Kazakhstan';
  portCount: number; // 1-1000
  format: 'login:password@host:port';
  rotation: 'none' | '5m' | '10m' | '30m';
}

// Генерация списка прокси через PROSOX API
// Хранение в Redis с TTL = период ротации
// Автосмена при 4xx/timeout
```

### 5.5 Fingerprint-стратегия

- **Провайдер:** FingerprintSwitcher (bablosoft) — до 100 бесплатных/день на MVP
- **Теги:** `['Kazakhstan', 'Chrome', 'Windows']` + `['Kazakhstan', 'Chrome', 'Android']` (мобильный трафик)
- Каждый профиль = уникальная связка {fingerprint + proxy + cookies}
- Профили хранятся в PostgreSQL, повторно используются для «постоянных посетителей»
- «Выгоревшие» профили автоматически заменяются (критерий: captcha или 3 неуспешных попытки)

### 5.6 Структура базы данных (основные сущности)

```
users — клиенты (email, тариф, баланс, созданный)
sites — сайты клиентов (домен, статус, гео)
keywords — ключевые слова (сайт, ключ, тип: накрутка/трекинг)
campaigns — кампании накрутки (настройки объёма, расписание)
visits — лог визитов (ключ, время, dwell, статус)
positions — история позиций (ключ, дата, позиция, URL)
proxies — пул прокси (строка подключения, провайдер, статус, использование)
fingerprints — пул профилей (JSON, статус, последнее использование)
transactions — биллинг (пользователь, сумма, тип, дата)
```

## 6. Безопасность и антидетект

| Мера | Реализация |
|------|-----------|
| KZ-гео синхронизация | timezone=Asia/Almaty + locale=ru-KZ + geolocation={43.25, 76.95} для Алматы |
| Рандомизация поведения | Bezier-кривые мыши, случайные паузы (1–5 сек), вариативный скролл |
| Ramp-up | Начало с 50 визитов/день, +10% ежедневно до лимита |
| Дневной лимит | Не более 10% от месячной частотности ключа |
| Блокировка медиа | `page.route('**/*.{png,jpg,gif,mp4}', route => route.abort())` — экономия трафика |
| Ротация прокси | Новый IP каждые 5–10 визитов (настраивается) |
| CDP-патчи | rebrowser-patches (убирает утечки Runtime.Enable) |
| CAPTCHA | Резолвер: 2Captcha или CapMonster (при необходимости) |
| Профили браузеров | Изолированные контексты, разные User-Data-Dir |

```
1 VPS: 16 GB RAM, 6 vCPU, Ubuntu 22.04
├── Docker Compose
│   ├── api (Node.js Fastify)

│   ├── worker (Playwright, 8–10 параллельных)
```

```
3–5 VPS (Hetzner / Contabo)
├── VPS-1: API + Frontend + PostgreSQL + Redis
├── VPS-2..5: Worker-фермы (по 10–20 браузеров каждая)

```
